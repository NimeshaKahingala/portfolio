# Portfolio Rebuild — Suggested Approach

> A ground-up redesign of [nimeshakahingala.com](https://nimeshakahingala.com) using a modern, minimal stack optimized for performance, SEO, and maintainability.

---

## Why Rebuild Instead of Refactor

The current codebase has accumulated enough structural debt that fixing it piece-by-piece would take longer than starting fresh:

- 3 competing styling systems (SCSS + Tailwind + CSS vars)
- CRA is deprecated — no path forward without ejecting
- ~13MB of unoptimized images bundled into JS
- antd (~300KB) imported for one modal
- Client-side rendering hurts SEO for a content site
- Dual header architecture (two separate components hidden via CSS)

A rebuild lets us solve all of these at once.

---

## Recommended Stack

| Layer | Technology | Why |
|---|---|---|
| **Framework** | Next.js 15 (App Router) | SSG for perfect SEO, file-based routing, built-in image optimization, API routes for contact form |
| **Styling** | Tailwind CSS v4 | Single styling system, CSS-first config, tiny output, design tokens built-in |
| **Icons** | Lucide React | Tree-shakeable, ~1KB per icon, modern consistent design |
| **Animations** | Framer Motion | Scroll-triggered reveals, page transitions, micro-interactions |
| **Contact Form** | Resend (via Next.js API route) | Free tier (100 emails/day), no mailto hack, real delivery confirmation |
| **Analytics** | Vercel Analytics (keep) | Already in use, zero-config with Vercel hosting |
| **Deployment** | Vercel (keep) | Already configured, perfect Next.js support |

### What Gets Removed

| Removed | Reason |
|---|---|
| Create React App (`react-scripts`) | Deprecated, slow builds |
| `antd` | 300KB for one modal → native `<dialog>` element |
| `sass` | Replaced by Tailwind |
| `react-helmet` | Replaced by Next.js built-in metadata API |
| `react-router-dom` | Replaced by Next.js file-based routing |
| `@fortawesome/fontawesome-svg-core` | Never used; Lucide replaces it |
| `web-vitals` / `reportWebVitals` | Vercel Speed Insights covers this |

### Key Structural Changes

1. **Content separated from code** — All project data, experience, skills live in `/content/*.json` files. Editing portfolio content never requires touching React components.
2. **Single header** — One `<Header>` component that is responsive via CSS breakpoints. No more rendering two headers and hiding one.
3. **Native `<dialog>` for modals** — Zero-dependency modal with built-in backdrop, focus trapping, and ESC-to-close.
4. **Images in `/public`, not bundled** — Project screenshots served as static files, optimized by Next.js `<Image>` component (auto WebP/AVIF, responsive sizes, lazy loading).
5. **File-based routing** — No router config file. Each `page.js` in `/app` is a route.
6. **Built-in 404** — `not-found.js` is automatic in Next.js App Router.

    title: 'Nimesha Kahingala — Full Stack Developer',
    description: 'Full Stack Developer specializing in React, Node.js, and AWS.',

- No client-side JS needed for meta tags
- Rendered at build time into static HTML
- Each page exports its own `metadata` object

### Auto-Generated Sitemap

```javascript

export default function sitemap() {
    return [
        { url: 'https://nimeshakahingala.com', lastModified: new Date() },
        { url: 'https://nimeshakahingala.com/about', lastModified: new Date() },
        // ...
    ];

```

No more manually maintaining `sitemap.xml`.

### Structured Data (JSON-LD)

Keep the existing Person schema from `index.html`, but move it to the root layout as a `<script>` tag rendered server-side.

### Keyword Stuffing Cleanup

Remove excessive `<strong>` tags around SEO keywords. Write naturally. Google's algorithms in 2026 penalize obvious keyword stuffing and reward natural, helpful content.

## Image Strategy

### Current Problem
- 28 files, ~13MB total
- All PNGs imported via JavaScript → bundled into Webpack chunks
- No responsive sizes, no modern formats

### Proposed Solution

```
public/images/
├── profile.webp              # Single profile photo (< 50KB)
└── projects/
    ├── millionspaces.webp     # Optimized to ~100-200KB each
    ├── golden-gate.webp
    ├── squadgurus.webp
    ├── 1billiontech.webp
    ├── banyan.webp
    ├── ceyphire.webp
    └── helpful-eng.webp
```

Use `next/image` component:
```jsx
<Image
    src="/images/projects/millionspaces.webp"
    alt="MillionSpaces workspace booking platform"
    width={800}
    height={450}
    placeholder="blur"
    blurDataURL="data:image/..."
    sizes="(max-width: 768px) 100vw, 50vw"
/>
```

Next.js automatically:
- Serves WebP/AVIF based on browser support
- Generates responsive sizes on demand
- Lazy loads below-the-fold images
- Adds blur placeholder while loading

**Expected total image size**: ~1-2MB (down from 13MB) — an **85% reduction**.

## Dark Mode Strategy

### Current
- Always starts as "light" mode
- User toggle stored in `localStorage`
- Theme applied via `data-theme` attribute

### Proposed
1. **Default to system preference** via `prefers-color-scheme` media query
2. **Manual override** persisted in `localStorage`  
3. **No flash of wrong theme** — use a `<script>` in `<head>` to set theme before paint

```javascript
// In root layout <head>:
<script dangerouslySetInnerHTML={{ __html: `
    const theme = localStorage.getItem('theme') 
        || (window.matchMedia('(prefers-color-scheme: dark)').matches ? 'dark' : 'light');
    document.documentElement.setAttribute('data-theme', theme);
`}} />
```

## Contact Form Implementation

### API Route (`app/api/contact/route.js`)

```javascript
import { Resend } from 'resend';

const resend = new Resend(process.env.RESEND_API_KEY);

export async function POST(request) {
    const { name, email, subject, message } = await request.json();

    // Server-side validation
    if (!name || !email || !message) {
        return Response.json({ error: 'Missing required fields' }, { status: 400 });
    }

    const { error } = await resend.emails.send({
        from: 'Portfolio Contact <contact@nimeshakahingala.com>',
        to: 'nimesha.isholi94@gmail.com',
        subject: `Portfolio Inquiry: ${subject}`,
        text: `From: ${name} (${email})\n\n${message}`,
    });

    if (error) {
        return Response.json({ error: 'Failed to send' }, { status: 500 });
    }

    return Response.json({ success: true });
}
```

### Client Component

```jsx
'use client';

const handleSubmit = async (e) => {
    e.preventDefault();
    setLoading(true);
    
    const res = await fetch('/api/contact', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(formData),
    });

    if (res.ok) {
        showToast('Message sent successfully!');
        resetForm();
    } else {
        showToast('Failed to send. Please try again.', 'error');
    }
    
    setLoading(false);
};
```

## Animation Strategy

Use **Framer Motion** for:

1. **Page transitions** — fade/slide between routes
2. **Scroll reveals** — elements animate in as they enter the viewport
3. **Hover micro-interactions** — card lifts, button pulses
4. **Modal open/close** — smooth scale + fade

### Reusable ScrollReveal Wrapper

```jsx
'use client';
import { motion } from 'framer-motion';

export default function ScrollReveal({ children, delay = 0 }) {
    return (
        <motion.div
            initial={{ opacity: 0, y: 30 }}
            whileInView={{ opacity: 1, y: 0 }}
            viewport={{ once: true, margin: '-100px' }}
            transition={{ duration: 0.6, delay }}
        >
            {children}
        </motion.div>
    );
}
```
